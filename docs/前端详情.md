# Just-We 前端详情

更新时间：2026-04-18

本文描述当前前端真实实现、页面职责、API 接线、状态管理和已知差距。当前前端主线是 Vue 3 + Vite + Element Plus + Pinia，页面已切换到 v2。

## 1. 技术栈与入口

| 项目 | 当前实现 |
| --- | --- |
| 框架 | Vue 3 |
| 构建 | Vite |
| UI | Element Plus |
| 状态管理 | Pinia |
| 样式 | SCSS、`frontend/src/assets/styles/v2/tokens.scss`、`global.scss` |
| HTTP | Axios 封装在 `frontend/src/utils/request.js` |
| 路由 | Vue Router，配置在 `frontend/src/router/index.js` |

入口文件：

- `frontend/src/main.js`：创建 Vue app、安装 Pinia、Router、Element Plus 和全局样式。
- `frontend/src/App.vue`：只渲染 `<router-view />`。
- `frontend/vite.config.js`：Vite 配置。

## 2. 全局请求与鉴权

HTTP 请求封装：

- 默认 `baseURL` 是 `VITE_API_BASE_URL`，未配置时为 `/api`。
- 请求拦截器从 `useAuthStore()` 读取 token 并写入 `Authorization: Bearer ...`。
- 响应拦截器遇到非登录请求 401 会清理 token 并跳转登录页。
- 其他错误统一通过 Element Plus message 展示后端 `detail` 或 `error`。

认证 store：

- 文件：`frontend/src/stores/auth.js`。
- token 存储在 `localStorage`。
- `login()` 调用 `POST /auth/login`，成功后再调用 `GET /auth/me`。
- `register()` 虽然封装存在，但新版 UI 不提供公开注册入口。
- `isAdmin` 由 `user.role === 'admin'` 计算。

权限 composable：

- 文件：`frontend/src/composables/usePermissions.js`。
- `admin` 可管理用户、系统、权重、日志。
- `admin` 和 `operator` 可管理抓取账号与监测对象。
- `viewer` 只应查看。

## 3. 路由结构

`/login` 使用独立登录页，不展示业务 shell。

登录后的根路由使用 `V2AppShell`，子路由如下：

| URL | 页面 | 权限 | 说明 |
| --- | --- | --- | --- |
| `/dashboard` | `DashboardV2.vue` | 登录用户 | 总览、抓取全部、健康摘要、作业、最新文章 |
| `/capture-accounts` | `CollectorAccountsV2.vue` | 登录用户；操作按钮按角色禁用 | 抓取账号、二维码、健康、fakeid、登录代理 |
| `/mp-accounts` | `MonitoredAccountsV2.vue` | 登录用户；操作按钮按角色禁用 | 监测对象、Feed、历史回填、Tier 编辑 |
| `/articles` | `ArticleListV2.vue` | 登录用户 | 文章筛选、详情入口、原文、重跑 AI |
| `/articles/:id` | `ArticleDetailV2.vue` | 登录用户 | 正文、图片、AI 三段结果、重跑 AI |
| `/proxies` | `ProxyManageV2.vue` | 登录用户 | 代理 CRUD、服务绑定、批量导入、测试 |
| `/logs` | `JobsLogsV2.vue` | admin | 作业与操作审计 |
| `/system-users` | `SystemUsersV2.vue` | admin | 用户管理 |
| `/weight` | `WeightConfigV2.vue` | admin | 权重配置与模拟 |
| `/settings` | `SystemSettingsV2.vue` | admin | AI、抓取、限频、代理、通知配置 |

兼容路由：

- `/accounts` 重定向到 `/mp-accounts`。

## 4. 全局 Shell

文件：`frontend/src/components/v2/V2AppShell.vue`。

职责：

- 顶部胶囊导航：总览、账号、监测、文章、代理、更多。
- 右侧主行动按钮：根据当前路由触发页面事件，例如抓取全部、新增账号、添加监测、导入代理、保存设置。
- 通知抽屉：读取 `useNotificationsStore()`，支持仅看未读、全部已读、单条已读、删除。
- 用户菜单：显示邮箱和角色，管理员可进入系统设置，所有用户可退出登录。

当前注意点：

- 通知跳转函数读取 `item.metadata_json`，但后端通知响应使用 `payload` 字段。需要统一字段名，否则资源跳转可能失效。
- 入口文案和页面 action rail 有较多功能说明文字，后续可减少解释性文案，保留状态和操作。

## 6. 页面详情

### 6.1 登录页

文件：`frontend/src/views/v2/LoginV2.vue`。

当前功能：

- 用户名/邮箱和密码登录。
- inline 错误提示。
- 登录成功后跳转 `redirect` 或 `/dashboard`。
- 不展示业务导航。

产品约束：

- 公开注册不作为主入口；用户应由管理员在系统用户页创建。

### 6.2 仪表盘

文件：`DashboardV2.vue`。

调用接口：

- `getCollectorAccounts()`。
- `getMonitoredAccounts()`。
- `getArticles({ page: 1, page_size: 20 })`。
- `getFetchJobs()`。
- `getProxyStats()`。
- `getLogStats()`。
- `triggerAllFetchTasks()`。

展示内容：

- 抓取账号健康数、监测对象数、近 24h 新文章、阻塞告警。
- 最近作业队列，前置失败分类、账号、代理、页码。
- 抓取账号健康摘要。
- 代理与日志摘要。
- 最新文章和 AI 命中标签。

### 6.3 抓取账号页

文件：`CollectorAccountsV2.vue`。

当前功能：

- 账号类型切换：`mp_admin`、`weread`。
- `mp_admin` 生成二维码前必须选择登录代理。
- 展示二维码、状态、有效期和错误消息。
- 每 3 秒轮询二维码状态。
- 账号列表按类型和健康状态筛选。
- 健康检查、fakeid 发现、更换登录代理、删除账号。

调用接口：

- `/collector-accounts/`。
- `/collector-accounts/qr/generate`。
- `/collector-accounts/qr/status`。
- `/collector-accounts/qr/{ticket}`。
- `/collector-accounts/{id}/health-check`。
- `/collector-accounts/{id}/discover-fakeid`。
- `/collector-accounts/{id}/login-proxy`。
- `/proxies/` 用于选择登录代理。

注意点：

- 登录代理筛选只接受 `service_keys` 包含 `mp_admin_login` 且类型是静态/长期代理。
- 更换登录代理会使凭证失效，页面已有风险提示。

### 6.4 公众号监测页

文件：`MonitoredAccountsV2.vue`。

当前功能：

- 通过公众号文章链接创建监测对象。
- 列表展示状态、fakeid、能力标签、Tier、分数、当前抓取方式、下次抓取。
- 复制单源 RSS/Atom/JSON Feed。
- 导出 OPML/CSV。
- 手动抓取、历史回填、回填状态、停止回填。
- 编辑名称、fakeid、状态、目标 Tier。
- 跳转文章列表并带 `monitored_account_id` query。

注意点：

- 后端聚合 Feed 接口存在，但当前页面复制聚合 Feed 从监测列表里找 `aggregate_feed_token`，而 `MonitoredAccountResponse` 不返回该字段。应改为使用 `authStore.user.aggregate_feed_token`。
- fakeid 不存在时页面会显示“仅详情抓取”，与后端能力标签一致。

### 6.5 文章列表页

文件：`ArticleListV2.vue`。

当前功能：

- 按公众号、日期范围请求后端筛选。
- 前端本地筛选标题关键词、目标命中、AI 状态、内容类型。
- 文章卡片展示标题、公众号/作者、发布时间、AI 目标命中、AI 状态、内容类型、抓取通道。
- 打开详情、打开原文、重跑 AI。

注意点：

- 标题关键词、AI 状态、目标命中、内容类型当前是前端本地筛选；如果文章量大，应下沉到后端分页筛选，否则分页后的本地筛选会影响结果完整性。

### 6.6 文章详情页

文件：`ArticleDetailV2.vue`。

当前功能：

- 展示标题、作者、发布时间、监测对象、内容类型、AI 命中。
- 正文 tab：清洗后的 `content_html`、纯文本 `content`、原始载荷。
- AI tab：文字解析、图片解析、类型判断、合并结果。
- 展示图片，优先本地化 `images`，否则原始 `original_images`。
- 重跑 AI。

安全约束：

- 默认只用 `content_html` 渲染富文本，不直接渲染 `raw_content`。

### 6.7 代理管理页

文件：`ProxyManageV2.vue`。

当前功能：

- 代理总数、启用、冷却、平均成功率。
- 按服务键筛选。
- 新增代理、批量导入代理。
- 展开行展示当前服务绑定、可编辑服务绑定、不兼容原因。
- 测试、启停、删除代理。

已知差距：

- 新增表单和批量导入中 `rotation_mode` 使用了 `static`、`rotating` 等值，后端枚举实际是 `fixed`、`sticky`、`round_robin`、`per_request`、`provider_auto`。这会导致创建/更新代理请求校验失败，应优先修复。
- 前端兼容性判断是本地简化版；后端已经返回 `compatible_service_keys` 和 `incompatible_reasons`，页面应优先使用后端响应。

### 6.8 作业与日志页

文件：`JobsLogsV2.vue`。

当前功能：

- 抓取作业 tab：类型、状态、监测对象、抓取账号、代理、页码、抓取/保存、失败分类、时间、payload。
- 操作审计 tab：动作、目标类型、目标 ID、详情、结果。
- 支持按 action、target_type、monitored_account_id 筛选审计日志。
- payload 抽屉格式化 JSON。

当前边界：

- 页面 action rail 提到“停止历史回填 / 重试失败”，但该页目前没有直接按钮；停止回填入口在监测对象页。

### 6.9 系统用户页

文件：`SystemUsersV2.vue`。

当前功能：

- 管理员创建用户、编辑邮箱/角色/启用状态、删除用户。
- 角色展示：admin、operator、viewer。
- 高风险提示。

注意点：

- 页面显示最近登录字段使用 `last_login_at`，后端 `UserResponse` 字段是 `last_login`。应统一字段名。
- 页面 action rail 提到“重置密码”，当前页面和后端用户接口没有实现重置密码。

### 6.10 权重页

文件：`WeightConfigV2.vue`。

当前功能：

- 读取和保存权重配置。
- 编辑频率、时效、目标命中、稳定性权重。
- 编辑 Tier 阈值 JSON、抓取间隔 JSON。
- 模拟目标命中“是/不是”对 Tier 和得分的影响。

注意点：

- `tierThresholdsText` 从后端 `tier_thresholds` 列表生成，保存时也按数组下标映射到后端字段。
- 目标命中“是”映射为 1，“不是”映射为 0，和后端 AI 语义一致。

### 6.11 系统设置页

文件：`SystemSettingsV2.vue`。

当前功能：

- AI 文字解析、AI 图片解析、类型判断 Prompt。
- 抓取策略、Tier 抓取方式、历史回填最大页数。
- 代理策略和限频策略。
- SMTP 邮件和 Webhook 通知。
- 默认管理员提示。

调用接口：

- `/system/ai-config`。
- `/system/fetch-policy`。
- `/system/rate-limit-policy`。
- `/system/proxy-policy`。
- `/system/notification-email`。
- `/system/notification-policy`。

注意点：

- `updateFetchPolicy()` 当前后端忽略 `rate_limit_policy` 和 `notification_policy`，页面已经通过独立接口保存这两项，这是正确接线。

## 7. 视觉系统

当前 v2 使用：

- 浅雾背景、深色胶囊导航、柔黄主行动按钮。
- 组件在 `frontend/src/components/v2/`：
  - `V2Page`
  - `V2Section`
  - `V2MetricCard`
  - `V2StatusPill`
  - `V2ActionRail`
  - `V2Empty`

主要样式源：

- `frontend/src/assets/styles/v2/tokens.scss`
- `frontend/src/assets/styles/v2/global.scss`
- 各页面 scoped SCSS。

设计期望：

- 保持统一 v2 组件，不恢复已删除的旧 layout 和旧页面体系。
- 高风险操作直接在按钮附近提示，不依赖隐藏说明。
- 后续应减少装饰性说明文字，让页面更偏操作台。

## 8. 前端待办

优先级高：

- 修复代理 `rotation_mode` 枚举值。
- 修复通知跳转字段 `metadata_json` / `payload` 不一致。
- 修复系统用户最近登录字段 `last_login_at` / `last_login` 不一致。
- 聚合 Feed 复制改用当前用户 `aggregate_feed_token`。

优先级中：

- 将文章关键词、AI 状态、目标命中、内容类型筛选下沉到后端。
- 作业页补重试失败和停止历史回填直接入口，或删除未实现文案。
- 为系统设置增加测试 AI、测试邮件、测试 Webhook 的明确入口。

优先级低：

- 增加 Playwright smoke 测试。
- 抽出表单枚举常量，避免前后端枚举漂移。
- 精简 action rail 文案。
